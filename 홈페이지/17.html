2. LANG 환경변수 내용이 아래와 같을 때 date 명령 실행 결과
화면에 출력되는 내용을 각각 기록하라. 결과가 달라진 이유도 설명하라.
a) ko_KR.UTF-8 

minji822@cegw:~$ echo $LANG
ko_KR.UTF-8
minji822@cegw:~$ date
2019. 05. 17. (금) 10:25:00 KST

b) en_US.UTF-8

minji822@cegw:~$ LANG=en_US.UTF-8
minji822@cegw:~$ echo $LANG
en_US.UTF-8
minji822@cegw:~$ date
Fri May 17 10:26:00 KST 2019

달라진 이유 :  lang 환경변수에 따라 다른 결과가 나온다. 

3. 새로운 환경변수 DEPT 를 만들라. 이 변수에는 ComputerEngineering 이라는
내용이 들어가야 한다. echo 및 printenv 명령을 각각 사용하여 이 환경변수가
제대로 설정되었는지의 여부를 알아보아라.
minji822@cegw:~$ DEPT=ComputerEngineering
minji822@cegw:~$ set | grep DEPT
DEPT=ComputerEngineering
이곳에서 설정된 환경변수는 쉘 안에서만 사용가능함.(printenv시에 나타나지 않음)
외부에서도 사용가능하게 하기 위해서는 export 라는 명령을 내려 주어야 함(printenv시에 나타남)

4. PATH 환경변수는 어떤 역할을 하는가?
디렉터리 경로의 목록 사용자가 전체경로를 지정하지 않고 명령을 입력하면 해당 명령어가 어느 경로에 속하는지 알려줌

5. CLASSPATH 환경변수는 어떤 역할을 하는가?
클래스를 찾기 위한 경로,즉, JVM이 프로그램을 실행할 때, 클래스파일을 찾는 데 기준이 되는 파일 경로를 말하는 것이다.

6. 다음 문장의 의미는 무엇인가?
a) PATH=.:$PATH 
path를 재설정, 경로명에 . (현재디렉토리)를 추가

b) PATH=$PATH:.
현재경로명에 . 추가

8. 위 문제에 이어 PATH 환경변수의 내용을 모두 삭제한 후 다음 명령을
내려보아라. 어떤 결과가 나오는가? 이런 결과가 나온 이유는 무엇인가?
a) ls
minji822@cegw:~/hw$ ls
-bash: ls: 그런 파일이나 디렉터리가 없습니다
PATH 안의 내용을 모두 지웠기 때문에 ls 명령을 실행하여도 나오지 않음

b) /bin/ls 
minji822@cegw:~/hw$ /bin/ls
1 2 3 Test.class  test.java  hw0510  main main.c
이유 :  원래의 ls 명령은 bin 밑에 있기 때문에 bin 밑에서 실행시키면 원래대로 실행됨

c) cd / 
minji822@cegw:~/hw$ cd /
minji822@cegw:/$
이유: 현재 위치를 홈디렉토리로 옮겼기때문

e) printenv
minji822@cegw:/$ printenv
-bash: printenv: 그런 파일이나 디렉터리가 없습니다
